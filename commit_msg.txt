fix: Supabase Vault RPC calls failing over PostgREST

- Added `vault_wrappers.sql` containing `public.create_vault_secret` and `public.update_vault_secret`.
- Supabase by default does not expose the `vault` schema to PostgREST. Calling `.schema('vault').rpc('create_secret')` directly from the Next.js API route throws an error because the function cannot be found over HTTP.
- Updated `src/app/api/kitchen/credentials/route.ts` to call the public wrapper functions using the `service_role` client.
